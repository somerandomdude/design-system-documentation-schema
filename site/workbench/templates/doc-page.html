Key idea: A document holds one or more entities. Each entity has metadata and document blocks.

Document structure

A DSDS file is a JSON object. It needs a dsdsVersion and one of two shapes: an entity object or an entityGroups array.

{ "dsdsVersion": "0.15.1", "entity": { "kind": "component", "identifier": "button", "name": "Button" } }

Common entity properties

These properties appear on every entity type.

Entity kind discriminator. Machine-readable id.
Pattern: ^[a-z][a-z0-9-]*$ Human-readable name. What the entity is and does. Typed document block objects.

Entity kinds

Eight kinds cover the surface of a design system: component, token, pattern, and guide among them.

stable experimental draft deprecated
